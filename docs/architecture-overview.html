Section 3

Build Process — Step by Step

1. Config Files

package.json, tsconfig.json, tailwind.config.ts, next.config.js, postcss.config

2. Prisma Schema

10 models, enums (Role, StatusValue), all relations and indexes defined

3. Auth System

NextAuth credentials provider, JWT strategy, middleware route guards, role-based access

4. Lib Utilities

Zod validators, stage derivation logic, permission helpers, date/currency formatters

5. UI Components

shadcn/ui primitives: Button, Input, Dialog, Select, Popover, Badge, Table, Tabs, Toast

6. Server Actions

5 action files: asset CRUD, tracking management, comments, admin ops, saved views

7. Application Pages

Login, Dashboard, Asset Detail, Admin/Users, Admin/Stages — all with layouts and loading states

8. Dashboard Components

Pipeline table, stage cells, column filters, detail drawer, KPI strip with deal counts

9. Seed Data

3 users, 5 pipeline stages, 1 asset, 18 companies, 90 stage statuses, 25 comments

10. Docker + Supabase

docker-compose.yml, SQL generation via prisma migrate, .env configuration

11. GitHub + Vercel Deployment

Repository created, Vercel project linked, auto-deploy on push to main
