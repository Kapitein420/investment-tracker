Two Portals

Admin Portal

Routes: / /assets/[id] /admin/*

  • Pipeline table with inline stage editing
  • Stage summary KPIs + Pipeline Overview funnel
  • Document upload + signing link generation
  • NDA approval workflow
  • User management and stage configuration
  • Investor invitations with login credentials
  • Timeline view of all activity
  • CSV export of pipeline data

Investor Portal

Routes: /portal /portal/[assetId]

  • Deal dashboard showing active opportunities
  • Guided step-by-step journey (Teaser, NDA, IM, Viewing, NBO)
  • Inline document signing (modal with PDF + signature pad)
  • Download signed documents
  • Gated content: IM unlocks only after NDA approval
  • Progress tracking per deal

Security Architecture

Authentication

  • NextAuth JWT sessions with bcrypt password hashing
  • Credential-based provider (email + password)
  • Session includes: userId, role, companyId
  • Middleware route protection on every request

Authorization

  • 4 roles: ADMIN, EDITOR, VIEWER, INVESTOR
  • Server-side requireRole() check on all mutations
  • INVESTOR isolation: only sees own company data
  • Role-based routing in middleware

Data Validation

  • Zod schemas validate all Server Action inputs
  • Shared schemas between client forms and server
  • Type-safe end-to-end with TypeScript + Prisma

Document Security

  • Private Supabase Storage bucket (no public access)
  • Signed URLs with 2-hour expiry for document access
  • Signing tokens: one-time use, 30-day expiry
  • NDA approval gating via approvedAt field
