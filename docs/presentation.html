Investment Tracker

How the deal-pipeline dashboard was built — the tools, the connections, and how it's secured.

Prepared for the CTO & Development Team

● Live in production on Vercel
The What

What we built

An internal web app that tracks commercial real-estate deals through their pipeline — replacing static Power BI dashboards with a fully editable, live operational tool.

📊Live pipeline

Companies move through stages — Teaser, NDA, IM, Viewing, NBO — editable inline from the table.

🔁Always current

Comments, audit history, stale-row highlighting and KPI cards keep the team aligned in real time.

🔐Role-based

Admin, Editor, Viewer and Investor each see only what they should. Investors get a separate portal.

The How

How it's built

One modern, type-safe stack — front-end and back-end in the same codebase.

⚛️Front-end

Next.js 14 (App Router) · TypeScript · Tailwind CSS + shadcn/ui · TanStack Table · React Hook Form + Zod validation.

🗄️Back-end & data

Server Actions (no separate API layer) · Prisma ORM · PostgreSQL on Supabase · NextAuth for sessions.

📄Documents

In-app NDA generation & e-signing — pdf-lib, jsPDF and pdf.js build and sign PDFs server-side.

✉️Communication

Mailgun sends investor invites, NDA approvals and password resets via transactional email.

The Connections

External services

Three managed services, each with a single clear job.

▲Vercel

Hosting & deploys. Builds from GitHub on every push, serves the app globally, holds all production secrets as env vars.

⬡Supabase

Database & storage. Managed PostgreSQL (pooled connections) plus file storage for signed NDA documents.

✉Mailgun

Email delivery. EU region. Sends every transactional email the workflow needs, from invites to resets.

Source lives in GitHub → auto-deployed by Vercel. No manual servers to manage.

The Security

Secured in layers

1AuthenticationNextAuth signs a session JWT with a server-only secret · passwords hashed with bcrypt · CSRF + login rate-limiting built in.
2AuthorizationMiddleware checks the role on every request — Admin / Editor / Viewer / Investor routes are gated and isolated.
3DatabasePrisma parameterized queries (no SQL injection) · Supabase Row-Level Security, deny-all-to-anon on 14 tables as defense in depth.
4Transport & browserHSTS, X-Frame-Options DENY, nosniff, strict referrer & permissions policy — set on every response.
5SecretsEvery key read from Vercel env vars at runtime — never in git, never in the browser bundle, never hardcoded.
The Security

Keys can't leak

Five independent layers stop a secret ever reaching a public commit.

LayerWhat it blocks
.gitignore covers all .env filesStaging a secret file locally
Zero hardcoded secrets in sourceAccidental check-in via code
GitHub Push ProtectionA push that contains a known key shape
GitHub Secret ScanningAlerts instantly if anything slips through
Provider scanning (Vercel / Supabase / Mailgun)Auto-disables a leaked key upstream

Git history audited — only .env.example (a template) was ever committed. Clean.
