Project Report & Decision Log

Investment Tracker — Internal Deal Pipeline & Investor Portal

01

Executive Summary

The Investment Tracker is a full-stack deal pipeline management tool built to replace Power BI for internal deal tracking workflows. Power BI was limited to read-only dashboards and could not support CRUD operations, document signing, or investor-facing workflows.

This application delivers a complete admin pipeline interface with stage tracking, document management, and CSV import, alongside a dedicated investor portal with NDA signing, deal journey views, and Information Memorandum access. The system is live on Vercel and connected to a Supabase PostgreSQL database.

02

Architecture Decisions

Why we chose what we chose. Each decision was evaluated against alternatives with specific trade-offs in mind.

DecisionChoiceWhyAlternatives Considered
Framework Next.js 14 (App Router) Full-stack in one codebase. Server Actions eliminate the API layer. SSR for fast loads. Vercel-native deployment. Express + React (more complexity), Remix (smaller ecosystem)
Database PostgreSQL via Supabase Managed hosting with free tier. Built-in storage and auth options. Connection pooling for serverless environments. Firebase (NoSQL doesn't fit relational data), PlanetScale (MySQL, fewer features)
ORM Prisma Type-safe queries with auto-generated client. Migration system. Excellent developer experience. Drizzle (newer, less mature), raw SQL (no type safety)
Auth NextAuth with JWT Lightweight credential-based auth. Role system with middleware integration. No external dependency. Supabase Auth (adds dependency), Auth0 (overkill for internal tool)
UI Tailwind + shadcn/ui Utility-first CSS with accessible components. No vendor lock-in. Highly customizable. Material UI (heavy, opinionated), Chakra (less customizable)
Table TanStack Table Headless library with no forced UI. Sorting, filtering, and pagination built-in. AG Grid (paid for features), DataGrid (too heavy)
PDF Signing pdf-lib Zero dependencies. Works in Node.js. Embeds images and text directly into PDFs. PDFKit (more complex), DocuSign (paid, overkill)
Email Resend Modern API with free tier. Custom domain support. Great developer experience. SendGrid (more complex), AWS SES (requires AWS setup)
Hosting Vercel Zero-config for Next.js. Auto-deploy from GitHub. Preview URLs. Edge functions. AWS (complex), Railway (less Next.js optimized)
Storage Supabase Storage (private) Same platform as DB. Private bucket with signed URLs. Free tier included. AWS S3 (separate service), Cloudflare R2 (extra setup)

Data Model Explained

14 tables, each serving a specific purpose in the deal lifecycle.

User

Authentication and role-based access control. Supports 4 roles: ADMIN, MANAGER, ANALYST, INVESTOR.

Asset

The property or investment being sold. Central entity that all deal activity connects to.

Company

Investor, broker, or advisor entities. Represents organizations participating in deals.

PipelineStage

Configurable deal stages (not hardcoded). Allows the pipeline workflow to be customized without code changes.

AssetCompanyTracking

The core junction table: one company's journey through one asset's pipeline. Links Company + Asset + Stage.

StageStatus

Per-stage status tracking, kept separate from the tracking row for flexibility in status management.

Comment

Internal notes attached per tracking row. Enables team communication within deal context.

StageHistory

Full audit trail for all stage changes. Records who moved what, when, and from which stage.

ActivityLog

System-wide event log capturing all significant actions across the platform.

SavedView

User-saved filter configurations for the pipeline table. Persists custom views.

Document

Uploaded PDFs linked to specific stages. Supports NDA, LOI, and other deal documents.

SigningToken

One-time-use tokens for public signing pages. Enables external parties to sign without accounts.

InvestorInvite

Email invitation tracking. Records sent invites and their acceptance status.

AssetContent

Information Memorandum materials (PDF or landing page) per asset. Investor-facing deal content.

Integration Guide for CTO

How to integrate this tool into the main DILS system.

The application is fully self-contained: Next.js frontend + Prisma backend + Supabase PostgreSQL database. No external service dependencies beyond email (Resend) and storage (Supabase).

1

Deployment Strategy

Embed as a subdomain (tracker.dils.com), iframe within the main DILS interface, or merge the codebase into the main DILS application directly.

2

API Extraction

All server actions can be extracted into REST endpoints if needed. Each action file maps cleanly to a set of API routes.

3

Database Migration

Can migrate from Supabase to any PostgreSQL instance by changing the DATABASE_URL environment variable. Prisma handles schema sync.

4

Authentication Swap

NextAuth can be replaced with DILS's existing auth system by swapping the provider configuration. The role system (ADMIN, MANAGER, ANALYST, INVESTOR) maps to any RBAC system.

5

Investor Portal as Micro-Frontend

The investor portal is a separate route group (/portal) that could become its own micro-frontend, deployed independently if needed.
