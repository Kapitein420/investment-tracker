Agent Types

Each agent type has a specific role. Briefing them well is the key skill.

🔍Explore Agent

Reads the codebase. No edits. Returns a structured analysis: file paths, line numbers, data flows.

"Find how auth works, what models exist, how signing connects."

📐Plan Agent

Designs an implementation strategy. No code. Returns architecture decisions, file lists, trade-off analysis.

"Plan PDF signing field placement. Evaluate 5 approaches."

🛠️Build Agent

Writes new files or modifies existing ones. Most common type. Briefed with exact files + patterns to use.

"Create these 5 server action files with these signatures."

🐛QA Agent

Pen tests, audits, finds bugs. Returns prioritized issue lists with attack scenarios and fix recommendations.

"Audit multi-tenant security. Test all server actions for IDOR."

🔧Fix Agent

Patches specific bugs from a list. Briefed with the exact issue, file, and proposed fix.

"Fix 5 critical security issues: tokens, IDOR, XSS, JWT, RLS."

📝Doc Agent

Generates HTML documentation, reports, dashboards. Stakeholder-ready output.

"Generate CTO report with decisions log and cost analysis."

Coordination Patterns

How the orchestrator manages multiple agents working simultaneously

Pattern 1: Parallel Execution

Launch 3-4 agents simultaneously, each on independent files. Wait for completion notifications. Saves 60-70% time vs sequential.

Pattern 2: Complete Briefing

Each agent gets full context — file paths, existing patterns, exact signatures, examples. Agents have no memory of prior conversations, so prompts are self-contained.

Pattern 3: Output Verification

After agent completes, orchestrator verifies the work — runs npm run build, checks for type errors, fixes coordination issues like Prisma client regeneration.

Pattern 4: Conflict Detection

When two agents touch the same file, orchestrator catches the merge issues. Example: PDF agent + Security agent both modified document-actions.ts. Caught and fixed manually.

Pattern 5: Audit Then Fix

QA agents run first to produce a prioritized bug list. Fix agents are then briefed with specific items from that list. Avoids guessing what to fix.

Pattern 6: Background Tasks

Long-running agents run in background while orchestrator handles other work. Tasks notify on completion. No idle waiting.

The Meta-Skill

The hardest part isn't the code — agents handle that. The hardest part is orchestration: knowing when to use which agent type, briefing them with enough context, verifying their output, catching conflicts between parallel work, and deciding when to commit and push.

This is what Noah is becoming. Not a developer who writes code, but a director who deploys teams of AI agents to build production systems. The instinct for product, security, and user experience that took years to develop is now the constraint — the code itself is no longer the bottleneck.
